# 產品需求文件 (PRD) - 好好唸

## 1. 產品概述
**好好唸** 是一套輕量化、免後端伺服器的網頁應用程式（SPA）。旨在幫助使用者將任何語言的文字，轉換為「其母語的近似發音標註」（例如：將英文 "Hello" 轉換為中文 "哈囉"），並支援該語言的語音朗讀，讓跨語言發音學習不再有門檻。

### 1.1 核心價值
*   **零門檻發音輔助**：利用 AI 轉換近似音，降低學習外語發音的恐懼感。
*   **高隱私與免註冊**：純前端設計，使用者的 API Key 僅儲存於本機瀏覽器（Local Storage）。
*   **易於部署與分享**：可直接部署於 GitHub Pages，即開即用。
*   **智慧配額自癒**：支援背景自動檢查與切換有配額的 Gemini 模型，避免免費版配額受限時的卡頓。
*   **直覺性**：透過系統語音朗讀，讓使用者建立聽覺與視覺的連結。
*   **趣味性**：透過中文譯音讓外文學習變得平易近人。

## 2. 使用者需求
*   **學習者**：想快速知道某個單字或句子的發音，但看不懂國際音標或該語言字母。
*   **旅遊者**：在國外需要快速模仿當地語言的基本發音。
*   **銀髮族**：需要更直覺、不費力的語言學習輔助工具。

## 3. 功能需求

### 3.1 文字輸入與語言偵測
*   使用者可輸入任意語言的文字（支持單行與多行）。
*   系統需透過 AI 自動偵測輸入文字的語言類型（BCP-47 代碼）。

### 3.2 中文譯音生成
*   系統需將輸入文字轉換為「中文譯音」（以最接近發音的漢字表示）。
*   **規則**：應優先考慮發音的準確性而非文字意涵。

### 3.3 自動語音朗讀 (TTS)
*   送出請求後，系統需自動觸發 Web Speech API 進行朗讀。
*   應根據偵測到的語言切換對應的語音引擎。
*   **優化**：在 API 請求前預喚醒語音引擎以避免瀏覽器封鎖。

### 3.4 極簡輸出介面
*   僅顯示「輸入原文」與「中文譯音」。
*   **語言規範**：介面所有文字（按鈕、提示、標籤）必須使用**繁體中文**。
*   **格式要求**：中文譯音：[結果]。

## 4. UI/UX 規範

### 4.1 視覺設計
*   **主色調**：淺粉紅色系 (Pink 50-600)。
*   **氛圍**：溫溫暖、親切、低壓力的設計風格。
*   **組件**：
    *   圓角設計 (Rounded-2xl)。
    *   陰影效果 (Shadow-xl) 提升質感。
    *   使用 Lucide-react 圖標增加直覺性（如：播放、音量）。

### 4.2 交互邏輯
1.  使用者輸入文字。
2.  點擊「朗讀」按鈕。
3.  按鈕進入 Loading 狀態。
4.  下方出現結果區域，同時電腦/手機發出聲音。
5.  結果區域提供一個隱藏或半透明的音量按鈕，供使用者重複聆聽。

## 5. 技術架構

### 5.1 前端框架
*   React: 用於狀態管理與組件化開發。
*   Tailwind CSS: 用於響應式與視覺樣式建構。

### 5.2 核心技術
*   **Gemini 2.5 Flash API**:
    *   處理自然語言理解 (NLU)。
    *   生成結構化 JSON 資料（含譯音與語言代碼）。
    *   實作 Exponential Backoff 機制以處理網路不穩定。
*   **Web Speech API**:
    *   調用使用者設備內建的語音合成引擎，確保無須外部流量即可朗讀。

### 5.3 錯誤處理機制
*   **401 錯誤**：提示使用者重新整理頁面以更新授權。
*   **解析錯誤**：自動清理 AI 回傳的 Markdown 標記，確保 JSON 穩定解析。

## 6. 未來擴展方向
*   **歷史紀錄**：儲存使用者曾翻譯過的譯音。
*   **自訂語速**：提供滑桿讓使用者調整朗讀快慢。
*   **多種譯音風格**：提供「注音版」或「諧音有趣版」等不同風格的譯音。
