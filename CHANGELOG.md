# 變更日誌 (CHANGELOG)

## [0.1.15] - 2026-03-20
### 新增
- **自主修復連線機制 (Autonomous Self-Healing)**：現在在設定中點擊「重新偵測可用模型」時，系統會自動對清單中的前幾個模型進行連線測試（Ping），並自動選取第一個連線正常且有配額（Quota）的模型。這能解決某些模型顯示可用但配額為 0 (RESOURCE_EXHAUSTED) 的問題。
- **即時連線品質狀態**：偵測時會顯示連線測試進度，確保使用者知道系統正在尋找最佳路徑。

### 修正
- **配額不足錯誤解決方案**：不再依賴單一預設模型，而是根據使用者帳號的實際配額動態選取。

## [0.1.14] - 2026-03-20
### 新增
- **預設模型切換至 Gemini 2.0 Flash**：根據金鑰實測，此模型在目前區域的相容性最高，可大幅降低 `NOT_FOUND` 錯誤。
- **連線狀態透明化**：主介面的「API 已就緒」現在會直接顯示目前正在使用的模型名稱，例如：`● API 就緒 (gemini-2.0-flash)`。

### 修正
- **硬編碼模型失效問題**：徹底移除對舊版 `gemini-1.5-flash` 的預設依賴。
